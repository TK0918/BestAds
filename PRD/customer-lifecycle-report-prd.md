## 客户生命周期报表 PRD

### 1. 需求背景&目标

#### 1.1 需求背景
- 当前团队缺少统一的客户生命周期诊断视图, 难以快速判断客户卡在哪个事件, 以及卡住了多久.
- 针对单个客户排查时, 需要跨多个页面和多种口径手工比对, 效率低且容易漏看.
- 针对BD客户池管理时, 缺少统一的事件达成占比, 转化漏斗, 以及事件间耗时分布, 难以量化客户转化质量.

#### 1.2 需求目标
- 支持单客户分析: 展示客户在9个生命周期事件上的达成情况, 时间轴, 卡点与关键耗时, 以及消耗强度指标.
- 支持多客户分析: 在注册时间范围内, 输出事件达成占比(饼图), 事件A->事件B耗时分布(可配置), 生命周期转化漏斗, 首次账户消耗耗时分布, 以及客户明细列表.
- 统一生命周期事件口径, 确保页面所有图表与表格使用同一套定义.

#### 1.3 成功标准
- 单客户视图在1屏内可看到: 当前卡点, 卡点停留天数, 下户->关键事件耗时, 以及核心消耗强度指标.
- 多客户视图可在同一页面完成: 事件占比, A->B耗时分布, 漏斗与分布, 并可通过筛选(注册时间范围, BD)快速对比不同客户池质量.
- 所有指标可通过抽样数据验证口径一致性.

### 4. 功能清单

#### 4.1 生命周期事件定义(9个事件)

| 事件序号 | 事件名称 | 字段key(前端) | 定义 |
|---|---|---|---|
| 1 | 注册 | `registeredAt` | 客户在系统创建成功日期 |
| 2 | 打款 | `paymentAt` | 客户首次完成有效打款入账日期 |
| 3 | 下户 | `onboardedAt` | 客户完成下户流程日期 |
| 4 | 首次账户充值 | `firstRechargeAt` | 客户名下任一广告账户首次完成充值日期 |
| 5 | 首次账户消耗 | `firstSpendAt` | 客户名下任一广告账户首次产生有效消耗日期 |
| 6 | 累计消耗达到1k | `cum1kAt` | 客户累计消耗首次达到或超过1k的日期 |
| 7 | 累计消耗达到5k | `cum5kAt` | 客户累计消耗首次达到或超过5k的日期 |
| 8 | 累计消耗达到10k | `cum10kAt` | 客户累计消耗首次达到或超过10k的日期 |
| 9 | 累计消耗达到100k | `cum100kAt` | 客户累计消耗首次达到或超过100k的日期 |

#### 4.2 页面功能清单(按模块)

- 查询区
  - **筛选**: 报表模式(单客户分析/多客户分析)
  - **筛选**: 注册时间范围(开始日期, 结束日期)
  - **筛选**: BD负责人
  - **筛选**: 客户(仅单客户分析展示, 支持全部/单个客户)
  - **操作**: 查询(刷新当前面板所有图表与表格)
- 单客户分析
  - **卡片**: 当前卡点
  - **卡片**: 卡点停留天数
  - **卡片**: 下户->首次账户充值耗时(天)
  - **卡片**: 下户->首次账户消耗耗时(天)
  - **卡片**: 历史最高月消耗
  - **卡片**: 历史最高周消耗
  - **卡片**: 历史最高日消耗
  - **卡片**: 最近7天平均消耗
  - **图表**: 生命周期时间轴(9个事件节点, 显示达成日期与节点间耗时)
  - **表格**: 单客户/全部客户明细(9事件日期 + 当前卡点)
- 多客户分析
  - **KPI**: 客户总数
  - **KPI**: 打款转化率
  - **KPI**: 首次账户消耗转化率
  - **KPI**: 累计10k转化率
  - **KPI**: 首次账户消耗P50(天)
  - **图表**: 当前事件占比饼图(注册时间范围内)
  - **图表**: 事件A->事件B时长占比(柱状分桶, A/B可配置, 默认下户->首次账户充值)
  - **图表**: 生命周期转化漏斗(9阶段)
  - **图表**: 首次账户消耗天数分布(D1/D3/D7/D14/D30累计占比)
  - **表格**: 多客户明细(注册, 打款, 首次账户消耗, 注册->首次账户消耗天数, 当前卡点)

### 5. 功能1详述: 查询区与筛选规则

#### 5.1 页面位置
- 菜单路径: `报表 -> 客户生命周期报表`
- 页面区域: 顶部查询区

#### 5.2 查询区字段表

| 字段名 | 控件类型 | 默认值 | 可选值/格式 | 生效范围 | 指标口径说明 |
|---|---|---|---|---|---|
| 报表模式 | 下拉框 | `single` | `single`, `multi` | 全页面 | 决定展示`单客户分析`或`多客户分析`面板 |
| 注册时间范围-开始 | 日期 | 最早注册日期 | `YYYY-MM-DD` | 全页面 | 客户池下界, 条件为`registeredAt >= startDate` |
| 注册时间范围-结束 | 日期 | 最晚注册日期 | `YYYY-MM-DD` | 全页面 | 客户池上界, 条件为`registeredAt <= endDate` |
| BD负责人 | 下拉框 | `全部` | `全部`+BD列表 | 全页面 | `全部`不筛选, 否则条件为`row.bd === bd` |
| 客户 | 下拉框 | `全部` | `全部`+客户列表 | 仅单客户分析 | `全部`显示所有行, 指定客户仅保留该客户 |
| 查询 | 按钮 | - | 点击触发 | 全页面 | 强制执行一次筛选与重绘 |

#### 5.3 刷新与联动
- 字段变更后即时刷新数据.
- 点击查询按钮后再次刷新, 保证用户显式确认操作.
- 切换`报表模式`时, 同步切换页面Tab状态.

### 6. 功能2详述: 单客户卡片指标

| 卡片名称 | 计算公式/规则 | 空值规则 | 说明 |
|---|---|---|---|
| 当前卡点 | 按事件顺序检查第一个未达成阶段 | 不为空 | 阶段值: 未打款/未下户/未首次账户充值/未首次账户消耗/未达累计1k/5k/10k/100k/已完成 |
| 卡点停留天数 | `diffDays(stageStartAt, endDate)` | `stageStartAt`为空时`-` | `stageStartAt`取当前卡点对应起始事件日期 |
| 下户->首次账户充值耗时 | `diffDays(onboardedAt, firstRechargeAt)` | 任一为空`-` | 单位: 天 |
| 下户->首次账户消耗耗时 | `diffDays(onboardedAt, firstSpendAt)` | 任一为空`-` | 单位: 天 |
| 历史最高月消耗 | `maxMonthSpend` | 为空`-` | 页面展示单位当前为`USD` |
| 历史最高周消耗 | `maxWeekSpend` | 为空`-` | 页面展示单位当前为`USD` |
| 历史最高日消耗 | `maxDaySpend` | 为空`-` | 页面展示单位当前为`USD` |
| 最近7天平均消耗 | `recent7dAvgSpend` | 为空`-` | 页面展示单位当前为`USD` |

### 7. 功能3详述: 单客户生命周期时间轴

| 项 | 定义 |
|---|---|
| 节点集合 | 固定9个事件: 注册, 打款, 下户, 首次账户充值, 首次账户消耗, 累计1k, 累计5k, 累计10k, 累计100k |
| 节点状态 | 有日期为已达成, 无日期为未达成 |
| 节点文案 | 显示事件名称 + 事件日期(未达成显示`未达成`) |
| 节点间耗时 | `diffDays(currEventAt, nextEventAt)`, 任一为空显示`-` |
| 可视规则 | 已达成节点使用高亮, 未达成使用灰态 |

### 10. 功能6详述: 当前事件占比饼图

| 维度 | 定义 |
|---|---|
| 统计对象 | 注册时间范围内且满足BD筛选的客户 |
| 归类规则 | 每个客户取`当前已达成最新事件`作为所属分组 |
| 事件集合 | 9个生命周期事件 |
| 计算公式 | `事件占比 = 当前事件为该事件的客户数 / total` |
| 展示内容 | 饼图扇区 + 事件图例(人数, 占比) |

### 11. 功能7详述: 事件A->事件B时长占比柱状图

#### 11.1 A/B选择字段表

| 字段名 | 默认值 | 可选值 | 指标口径 |
|---|---|---|---|
| 事件A | 下户(`onboardedAt`) | 9个事件 | 时长起点事件 |
| 事件B | 首次账户充值(`firstRechargeAt`) | 9个事件 | 时长终点事件 |

#### 11.2 分桶口径表

| 桶名称 | 天数范围 | 归桶条件 |
|---|---|---|
| 1天内 | 0-1天 | `0 <= diffDays(A,B) <= 1` |
| 3天内 | 2-3天 | `2 <= diffDays(A,B) <= 3` |
| 7天内 | 4-7天 | `4 <= diffDays(A,B) <= 7` |
| 14天内 | 8-14天 | `8 <= diffDays(A,B) <= 14` |
| 30天内 | 15-30天 | `15 <= diffDays(A,B) <= 30` |
| 30天以上 | 31天及以上 | `diffDays(A,B) >= 31` |

#### 11.3 样本与占比
- 可计算样本: `A非空 && B非空 && diffDays(A,B) >= 0`.
- 桶占比: `bucketCount / sampleCount`.
- 图表显示: 每桶人数和占比, 并显示当前可计算样本数.
