## 2. 需求背景与目标

### 2.1 需求背景

- 业务提出【P0】广告维度看板需求，希望从**媒体平台投放 Breakdown 维度**（版位、年龄、性别等）查看花费结构与效果效率，而非仅从客户、Campaign 等组织维度汇总。
- 现有「广告客户表现」报表解决的是组织维度汇总；本看板解决的是**投放标签维度拆解**，二者互补，不替代。
- 当前 Reporting 同步能力为 `日期 + Ad + 国家` 粒度；版位、年龄、性别等需扩展 Meta Insights Breakdown 拉数，技术上已确认可行。
- 数据来源为**广告实际投放成效**（Insights Breakdown），不是广告设置时的配置字段（如 Adset 勾选的版位列表）。

### 2.2 需求目标

- 在运营系统报表模块新增独立页面「**广告多维度效果看板**」。
- 支持按全局筛选限定数据范围后，在多个维度 Tab 下查看花费、花费占比及 CPM/CTR/CPC/CPA/ROAS。
- 每个维度 Tab 采用统一布局：KPI 卡 + 图表区 + 明细表 + 导出。
- V1.0 优先支持 **Meta** 平台；版位、年龄、性别、总览为 P0；其余维度按版本分期。

### 2.3 核心原则

| 规则 | 说明 |
| --- | --- |
| 数据性质 | 使用媒体 **Insights Breakdown 成效数据**，不使用 Adset 配置镜像替代 |
| 指标计算 | 比率类指标必须先 SUM 分子分母，再计算；禁止对行级 CPA/ROAS 求平均 |
| 花费占比 | 仅在**当前 Tab 维度枚举值**之间计算，跨 Tab 不加总 |
| 筛选与维度 | 筛选只限定范围；Tab 决定按什么 Breakdown 分组展示 |
| 权限 | 客户范围在服务端过滤；BD 默认仅看负责客户 |
| 导出 | 导出字段、排序、口径与当前 Tab 页面一致 |

### 2.4 与现有报表关系

| 模块 | 关系 |
| --- | --- |
| 广告客户表现-自定义 Tab | 组织维度汇总，与本看板互补 |
| 广告消耗情况 / 广告周数据 | 时间序列监控，本看板偏结构拆解 |
| 现有 Reporting（date+ad+country） | 国家 Breakdown 可复用；版位/年龄/性别需新增同步任务 |

## 5. 全局指标定义

以下指标在所有 Tab 明细表、图表、KPI 卡中口径一致（除非某 Tab 单独说明）。

### 5.1 基础度量（可 SUM）

| 指标名称 | 字段 key | 类型 | 说明 |
| --- | --- | --- | --- |
| 花费 | `spend` | amount | 广告消耗，统一币种 |
| 展示 | `impressions` | number | 展示次数 |
| 点击 | `clicks` | number | 点击次数（口径与现有 Reporting 一致） |
| 转化 | `conversions` | number | 与 `conversionEvent` 配置一致 |
| 转化价值 | `conversionValue` | amount | ROAS 分子 |

### 5.2 派生度量（聚合后计算）

| 指标名称 | 字段 key | 计算公式 | 空值规则 |
| --- | --- | --- | --- |
| 花费占比 | `spendShare` | `SUM(spend) / SUM(spend) OVER (当前 Tab 全部分组行)` | 展示为百分比，保留 2 位小数 |
| CPM | `cpm` | `SUM(spend) / SUM(impressions) × 1000` | 分母为 0 显示 `-` |
| CTR | `ctr` | `SUM(clicks) / SUM(impressions)` | 展示为 %，保留 2 位小数；分母为 0 显示 `-` |
| CPC | `cpc` | `SUM(spend) / SUM(clicks)` | 分母为 0 显示 `-` |
| CPA | `cpa` | `SUM(spend) / SUM(conversions)` | 分母为 0 显示 `-` |
| ROAS | `roas` | `SUM(conversionValue) / SUM(spend)` | 分母为 0 显示 `-`；保留 2 位小数 |

### 5.3 合计行规则

| 字段类型 | 合计行规则 |
| --- | --- |
| 维度列 | 展示「合计」 |
| spend、impressions、clicks、conversions、conversionValue | 对当前查询结果全量 SUM（不受分页影响） |
| spendShare | 固定为 `100%`（或 `-`，二选一，前后端统一即可） |
| CPM、CTR、CPC、CPA、ROAS | 用合计后的分子、分母按上表公式重算 |

### 5.4 数据延迟与空状态

| 场景 | 展示 |
| --- | --- |
| 无数据 | 文案：`暂无数据，请调整筛选条件后重试` |
| Breakdown 未开通 | 文案：`当前广告账户未开通该维度数据，请联系管理员或在 Meta Ads Manager 中开通` |
| 查询超过 90 天 | 拦截查询，提示缩小时间范围 |

## 7. 各维度报表详细说明

### 7.0 总览 Tab

#### 7.0.1 业务意义

进入看板后第一眼看到整体规模，以及「花费最集中在哪个维度」的提示，帮助用户决定优先进入哪个维度 Tab。

#### 7.0.2 数据来源

- 不单独做 Breakdown 聚合，由各 Breakdown 事实表在服务端分别 `GROUP BY breakdown_value` 后取 Top1，或在总览接口内并行聚合。
- V1.0 总览仅汇总**版位、年龄、性别**三个维度的 Top1 花费占比（因 V1.0 仅上线此三维度）。

#### 7.0.3 图表与展示形式

| 区块 | 图表类型 | 说明 |
| --- | --- | --- |
| KPI 卡 ×4 | 数字卡 | 总花费、总转化、整体 ROAS、整体 CPA |
| 维度花费集中度 | **横向条形图**（迷你） | 每个维度 1 根条，长度为该维度 Top1 值的 `spendShare`；Y 轴标签如「版位」「年龄」「性别」 |
| 无饼图/无明细级环形图 | - | 总览不做占比饼图，避免信息过载 |

**KPI 卡指标**

| 卡片 | 字段 | 计算 |
| --- | --- | --- |
| 总花费 | `spend` | 当前筛选下，任意一张 Breakdown 表按 ad 去重前 SUM(spend) 需与明细一致；实现上可用「无 breakdown」汇总表或取 placement 表按 date+ad 去重后 SUM |
| 总转化 | `conversions` | 同上 |
| 整体 ROAS | `roas` | `SUM(conversionValue) / SUM(spend)` |
| 整体 CPA | `cpa` | `SUM(spend) / SUM(conversions)` |

**维度集中度条形图**

| 项目 | 说明 |
| --- | --- |
| 维度 | 行：版位 / 年龄 / 性别；值：各维度花费最高枚举值的 `spendShare` |
| 指标 | `spendShare`（%） |
| 计算 | 对每个 `breakdown_type`，`MAX(spendShare)` 所在行；若并列取 spend 最大者 |
| 辅助文案 | 条形右侧展示，如「版位：instagram / reels 占 32%」 |

#### 7.0.4 明细表

总览 Tab **无**标准 11 列明细表；可选展示「各维度 Top3 简表」（V1.0 可做可不做，默认**不做**，仅 KPI + 迷你条）。

### 7.1 版位 Tab

#### 7.1.1 业务意义

查看预算在实际投放版位（Facebook Feed、Instagram Reels 等）的分布，以及各版位 CPM/CTR/ROAS 差异，支撑版位收缩或扩张决策。

#### 7.1.2 维度定义

| 项目 | 说明 |
| --- | --- |
| breakdown_type | `placement` |
| 分组主键 | `publisherPlatform` + `platformPosition` |
| 展示字段 `breakdownLabel` | `{publisherPlatform} / {platformPosition}`，如 `instagram / reels` |
| 标准枚举（归一后） | 见下表 |

**版位归一枚举（Meta）**

| publisher_platform | platform_position（示例） | breakdownLabel 展示 |
| --- | --- | --- |
| facebook | feed | Facebook / Feed |
| facebook | video_feeds | Facebook / Video Feeds |
| instagram | stream | Instagram / Feed |
| instagram | reels | Instagram / Reels |
| instagram | story | Instagram / Story |
| messenger | messenger_inbox | Messenger / Inbox |
| audience_network | classic | Audience Network / Classic |
| 其他 | - | Other |
| 未知 | - | Unknown |

> 原始值通过维表 `dim_placement_mapping` 映射为展示文案；未映射归入 Other。

#### 7.1.3 图表与展示形式

| 区块 | 图表类型 | 维度/指标 | 说明 |
| --- | --- | --- | --- |
| KPI 卡 ×4 | 数字卡 | 花费、花费占比 Top1 版位的 spend、整体 CPM、整体 CTR | Top1 指 spend 最大的 `breakdownLabel` |
| 图 1-花费结构 | **横向条形图** | X：`spend`；Y：`breakdownLabel` | 按 spend 降序；默认展示 Top15，其余合并为 Other |
| 图 2-花费占比 | **环形图（Donut）** | 值：`spendShare`；分类：`breakdownLabel` | 同 Top15 + Other 规则；中心可显示总花费 |
| 图 3-效率对比（可选第三图，V1.0 放在图 2 右侧） | **分组柱状图** | X：`breakdownLabel`（Top10）；系列：ROAS、CPA | 双 Y 轴或归一化展示，默认只显示 Top10 |

**是否使用饼图**：使用 **环形图** 仅表达花费占比；主对比仍用横向条形图（类目多时可读性更好）。

#### 7.1.4 明细表

- 列：遵循第 6 节通用列序。
- 默认排序：`spend` DESC。
- Other 行：花费占比 &lt; 1% 的枚举值合并为一行 `Other`，指标 SUM 后重算比率。

#### 7.1.5 指标计算示例

设筛选后版位 A、B 花费分别为 800、200，总花费 1000：

| 版位 | spend | spendShare | ROAS 计算 |
| --- | --- | --- | --- |
| A | 800 | 80% | 用 A 行 SUM(conversionValue)/SUM(spend) |
| B | 200 | 20% | 同上 |
| 合计 | 1000 | 100% | 用 1000 与合计 conversionValue 重算 |

### 7.5 视频时长 Tab（V1.1）

#### 7.5.1 业务意义

分析不同视频时长分桶的投放效果，指导创意时长策略。

#### 7.5.2 维度定义

| 项目 | 说明 |
| --- | --- |
| 数据性质 | **配置+关联**：Creative API 视频时长分桶后，与 ad 层 spend 关联（非 Meta 标准 Breakdown） |
| breakdown_type | `video_length` |
| breakdownLabel | `0-15s`、`15-35s`、`35-60s`、`60s+`、`Non-video`、`Unknown` |

**分桶规则**

```
duration_seconds <= 15  → 0-15s
15 < duration <= 35     → 15-35s
35 < duration <= 60     → 35-60s
duration > 60           → 60s+
无视频创意              → Non-video
无法解析                → Unknown
```

#### 7.5.3 图表与展示形式

| 区块 | 图表类型 | 说明 |
| --- | --- | --- |
| 图 1 | **横向条形图** | spend |

## A.3 Meta Insights 拉数规格（V1.0）

### A.3.1 公共参数

| 参数 | 值 |
| --- | --- |

## A.4 数仓聚合逻辑（看板查询）

以版位 Tab 为例，SQL 逻辑示意：

```sql
SELECT
  CONCAT(publisher_platform, ' / ', platform_position) AS breakdown_label,
  SUM(spend) AS spend,
  SUM(impressions) AS impressions,
  SUM(clicks) AS clicks,
  SUM(conversions) AS conversions,
  SUM(conversion_value) AS conversion_value
FROM dws_ad_breakdown_placement_daily t
JOIN dim_ad ad ON t.ad_id = ad.ad_id
WHERE t.stat_date BETWEEN :start AND :end
  AND ad.merchant_id IN (:merchantIds)  -- 权限+筛选
GROUP BY publisher_platform, platform_position
ORDER BY spend DESC;
```

占比在应用层或窗口函数计算：

```
spendShare = spend / SUM(spend) OVER ()
```

比率指标在应用层：

```
cpm = spend / impressions * 1000
ctr = clicks / impressions
roas = conversionValue / spend
```

## A.9 风险清单

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| Breakdown 数据量膨胀 | 同步慢、存储大 | 分区表、异步 Job、仅同步活跃账户 |
| 不同 Breakdown 花费之和≠总花费 | 业务误解 | 页脚说明「Breakdown 为分发视角，不宜与账户总花费直接对账」 |
| age/gender 账户未开通 | Tab 无数据 | 能力检测 + 开通指引 |
| 配置维度与 Breakdown 混用 | 产品预期偏差 | PRD 页脚区分「投放拆解」与「配置汇总」 |

## A.10 与现有 Reporting 的衔接

```
现有: ODS → dws (date, ad_id, country) → 广告客户表现/Reporting API

新增:
  ODS → dws_placement (date, ad_id, publisher_platform, platform_position)
  ODS → dws_age (date, ad_id, age)
  ODS → dws_gender (date, ad_id, gender)
       ↓
  AdDimensionDashboardQueryService
```

**复用**：账户授权、转化事件解析、异步 Job 框架、权限（merchant_id / BD 归属）与现有 Reporting 保持一致。
